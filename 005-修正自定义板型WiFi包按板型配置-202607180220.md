# 修正自定义板型 WiFi 包按板型配置

此前脚本对所有自定义板型统一移除了 NanoPC-T4 默认 Broadcom WiFi 包，这会导致后续自定义板如果使用同款 WiFi 芯片时无线不可用。

本次改为按板型配置 WiFi 包：

## dg3399

dg3399 使用 AP6212，对应 Broadcom BCM43430 SDIO。

构建时保留 `kmod-brcmfmac` 和 `wpad-openssl`，仅替换固件/NVRAM：

```text
-brcmfmac-firmware-4356-sdio
-brcmfmac-nvram-4356-sdio
brcmfmac-firmware-43430a0-sdio
brcmfmac-nvram-43430-sdio
```

## boocax

boocax 的 WiFi 与 NanoPC-T4 同型号，因此不额外调整 WiFi 包，继续沿用 NanoPC-T4 profile 默认包：

```text
kmod-brcmfmac
wpad-openssl
brcmfmac-firmware-4356-sdio
brcmfmac-nvram-4356-sdio
```

## 后续新增板型

新增板型时，不要直接复制 dg3399 的 WiFi 包配置。应按实际 WiFi 芯片设置 `CUSTOM_WIFI_PACKAGES`：

```bash
CUSTOM_WIFI_PACKAGES=""
```

表示沿用借用 profile 的默认 WiFi 包。

如果要替换固件，使用 ImageBuilder 的包语法：

```bash
CUSTOM_WIFI_PACKAGES="-旧固件包 -旧NVRAM包 新固件包 新NVRAM包"
```
