# dg3399 设备树替换说明

本项目使用 ImmortalWrt ImageBuilder 构建 Rockchip 镜像，当前上游构建选项中没有 dg3399 profile，因此采用 NanoPC-T4 profile 作为打包入口。

## 使用方式

在 GitHub Actions 的 Rockchip 构建页面中，`profile` 选择：

```text
friendlyarm_nanopc-t4
```

构建脚本会在执行 `make image` 前，把 ImageBuilder 内部找到的 `rk3399-nanopc-t4.dtb` 替换为本仓库的：

```text
rockchip/dtb/rk3399-dg3399.dtb
```

这样生成出来的镜像仍然使用 NanoPC-T4 的 profile 名称，但打包进镜像的设备树来自 dg3399。

## 已修改文件

- `rockchip/build24.sh`
- `rockchip/build25.sh`
- `.github/workflows/build-rockchip-immortalWrt-24.10.x.yml`
- `.github/workflows/build-rockchip-25.12.x.yml`
- `rockchip/dtb/rk3399-dg3399.dtb`

## 注意事项

- 该替换逻辑只在 `PROFILE=friendlyarm_nanopc-t4` 时启用。
- 如果 ImageBuilder 内部找不到 `rk3399-nanopc-t4.dtb`，构建会直接失败并打印部分 `rk3399*.dtb` 路径，方便定位上游路径变化。
- 本次没有执行编译动作，编译仍由你手动触发。
