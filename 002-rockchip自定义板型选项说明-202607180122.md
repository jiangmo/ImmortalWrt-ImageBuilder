# Rockchip 自定义板型选项说明

本次调整后，Rockchip 的 GitHub Actions 构建选项中新增两个自定义板型：

```text
dg3399
boocax
```

## 设备树文件

两个板型分别使用以下设备树文件：

```text
rockchip/dtb/rk3399-dg3399.dtb
rockchip/dtb/rk3399-boocax.dtb
```

## 构建逻辑

由于当前 ImmortalWrt ImageBuilder 没有原生提供 `dg3399` 和 `boocax` profile，脚本会在内部自动映射：

```text
dg3399  -> friendlyarm_nanopc-t4
boocax  -> friendlyarm_nanopc-t4
```

映射只作为 ImageBuilder 打包入口使用。当前实际打包方式以 `003-修正自定义设备树FIT打包方式-202607180147.md` 为准：脚本会覆盖 `make image` 的 `KERNEL` 管线，让 `fit lzma` 直接使用当前选择板型对应的 dtb。

## 使用方式

在 GitHub Actions 的 Rockchip 构建页面中，直接选择：

```text
dg3399
```

或：

```text
boocax
```

无需再手动选择 `friendlyarm_nanopc-t4`。

## 输出文件

自定义板型构建完成后，脚本会尝试把输出镜像文件名中的 `nanopc-t4` 替换成当前板型名，方便在 Release 附件中区分。

## 注意事项

- 该逻辑只影响 `dg3399` 和 `boocax` 两个自定义 profile。
- 其他 Rockchip profile 继续按 ImageBuilder 原生方式构建。
- 本次没有执行编译动作，编译仍由你在 GitHub Actions 中手动触发。
